Установка:

```bash
npm install --save-dev test-proxy-recorder
```

## Быстрее всего: сгенерировать через `init`

Одна команда подключает test-proxy-recorder к проекту:

```bash
npx test-proxy-recorder init http://localhost:3002 --port 8100 --dir ./e2e/recordings
```

Все аргументы необязательны и откатываются к разумным значениям по умолчанию (`http://localhost:3000`, порт `8100`, `./e2e/recordings`). Файлы создаются и изменяются **без разрушения** — существующие файлы и скрипты никогда не перезаписываются, если вы не передали `--force`.

### Что генерирует и изменяет `init`

- `test-proxy-recorder.config.ts` — конфигурация прокси (обнаруживается автоматически, поэтому затем `npx test-proxy-recorder` не требует флагов).
- `playwright.config.ts` — добавляет `webServer`, указывающий на эндпоинт `/__control` прокси, плюс `globalTeardown`. Существующая конфигурация Playwright **редактируется на месте**; если Playwright нет вовсе, `init` сначала запускает CLI Playwright для настройки (передайте `--no-install`, чтобы пропустить).
- `e2e/fixtures.ts` и `e2e/global-teardown.ts` — фикстура прокси для каждого теста и teardown.
- `package.json` — добавляет скрипты `proxy`, `proxy:reset`, `test:e2e` и `test:e2e:record`. Если у вас есть скрипт `dev`, он оборачивается: исходный переезжает в `dev:app`, а `dev` становится командой `concurrently`, которая запускает прокси рядом с вашим приложением (так `npm run dev` записывает во время разработки). `concurrently` добавляется в `devDependencies`.

Конфигурация Playwright, в которой уже определён `webServer`, остаётся нетронутой, с заметкой о том, что нужно добавить.

## Единственный ручной шаг

**Единственный шаг, который `init` не может сделать за вас**, — направить вызовы бэкенда вашего приложения через прокси: какая переменная окружения хранит базовый URL вашего API и как ограничить это режимом разработки, зависит от приложения. По завершении `init` печатает конкретные инструкции для этого: направьте эту переменную окружения на `http://localhost:8100` **только в dev/test, никогда в продакшене** (например, добавьте префикс к скрипту `dev:app`, используя `cross-env` в Windows). Затем прокси пересылает запросы на ваш реальный бэкенд во время записи и отдаёт записи при воспроизведении.

Затем напишите тест, запишите один раз против реального API, переключитесь на воспроизведение и закоммитьте `e2e/recordings/`. [Ручная настройка](/ru/docs/getting-started/manual-setup/) показывает этот цикл целиком.
